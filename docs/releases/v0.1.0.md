### Core Features

#### 1. Pure Function Orchestrator

```typescript

const result = await runLlmStep(
  form,          // FormDefinition
  session,       // Session state

// - botResponse: Natural language response

// - nextField: Suggested next field to collect
```

## Quick Start

```typescript
import {
  runLlmStep,
  type FormDefinition,
  type Session,
  type LlmClient,
  FieldType,
  SessionStatus,
} from '@flowform/core';

// 1. Define your form
const form: FormDefinition = {
  id: 'contact',
  name: 'Contact Form',
  fields: [
    {
      id: 'name',
      name: 'name',
      label: 'Name',
      type: FieldType.TEXT,
      required: true,
      order: 0,
    },
    {
      id: 'email',
      name: 'email',
      label: 'Email',
      type: FieldType.EMAIL,
      required: true,
      order: 1,
    },
  ],
  createdAt: new Date(),
  updatedAt: new Date(),
};

// 2. Initialize session
const session: Session = {
  id: 'session-1',
  formId: 'contact',
  status: SessionStatus.ACTIVE,
  turns: [],
  fields: [],
  startedAt: new Date(),
};

// 3. Implement LLM client
const llmClient: LlmClient = {
  async complete(messages) {
    // Your LLM provider here (OpenAI, Anthropic, etc.)
    const response = await yourProvider.complete(messages);
    return { content: response };
  },
};

// 4. Process user input
const result = await runLlmStep(
  form,
  session,
  'My name is John and email is john@example.com',
  llmClient
);

console.log(result.botResponse);      // "Thanks John! Is that all?"
console.log(result.extractedFields);  // { name: "John", email: "john@example.com" }
console.log(result.isComplete);       // true
```

## Architecture

The orchestrator follows **Clean Architecture** principles:

- **Pure Functions** - No side effects, fully testable
- **Dependency Injection** - LLM client injected for easy mocking
- **Zero Dependencies** - No external runtime dependencies
- **Framework Agnostic** - Works with any UI or backend framework
- **Type Safe** - Strict TypeScript with discriminated unions
